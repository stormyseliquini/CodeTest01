How do you define an AngularJS application? An application written in javascript that is wrapped in the robust angular framework. This framework is designed to be more functional in larger projects, by providing faster speeds and more scalability.

What is a controller? A controller is a constructor function used to bind the view to the application.

What does ng-model do? It binds an output from the application to the view, similar to document.getElementById(id).innerHTML in stock javascript, and $(id).text in jquery.

What is interpolation markup? The syntax used to inject the value of an expression (as opposed to the expression itself) into the view.

What is $scope? An object that binds the html to the controller.

What is dependency injection? A tool to inject a dependency object from outside of the scope, into it, as opposed to writing the logic for a dependency within the scope itself.

What is two-way binding? Simultaneously, when a change to the view automatically changes the model, and a change to the model automatically changes the view.

What is 'controller as' syntax? It is syntax used to circumvent the $scope object and prevent scope bleed.

What is MV* architecture? The previous best-practice architecture of Angular came with many disadvanteges that could've potentially, in large scale applications, outweighed the advantages of the Angular framework altogether. This is because of the manner of which the model and view were binded via the controller. In early versions of Angular, the controller consisted of heavy logic and the $scope object, which lead to a bloated application and scope bleed. With the vm* architecture, a controller is converted into a directive, ideally with one controller for one html template. What this does is help eliminate the need for logic within a controller almost entirely, and allow for easier unit testing, reuse of code and implimentation of code.

What is the difference between a service and a factory? At the most basic level, both a service and a factory accept a name and a function, the differene being that a service is a constructor function and a factory is not. A factory returns an object literal and can be reinstantiated, while a service is really designed to be used once. However, these differences aren't highly apparent in practice; in fact, if desired a service can do exactly what a factory can do. Furthermore, a service actually is just calling upon a predefined factory with the angular source code, as opposed to creating one.
